# 2026-04-26 - ClawPass Sync Investigation

## Session Focus
Working on ClawPass iOS sync functionality - specifically entry merge logic so entries received from desktop sync are properly merged into the iOS vault.

## Critical Findings

### Root Cause of Broken Sync
The desktop TCP sync server has **TWO SEPARATE VAULTS** that are never synchronized:

1. **AppState.vault** (Tauri state) - Populated when vault is unlocked via `unlock_vault` command. Contains actual entries.
2. **sync_tcp's vault** - `Arc<RwLock<Vec<VaultEntry>>>` initialized as empty `Vec::new()` in main.rs. Never gets populated!

```rust
// main.rs creates EMPTY separate vault for sync_tcp
let vault = Arc::new(RwLock::new(Vec::<VaultEntry>::new()));
let vault_clone = vault.clone();
sync_tcp::TcpSyncServer::start(vault_clone, 7878)  // sync_tcp uses this empty vault
```

When iOS requests sync → desktop returns empty entries because it's reading from the empty sync_tcp vault.

### What Was Already Implemented (Desktop)
- `sync_tcp::TcpSyncServer::start_with_state()` - Already exists in sync_tcp.rs! Syncs vault from AppState to TCP server when client connects
- BUT main.rs uses `start()` instead of `start_with_state()` - The functionality exists but isn't being used!

### iOS Issues (VaultManager.swift)
- `syncService(_:didReceiveEntries:)` only **logs** received entries, doesn't merge them
- No delegate method for `EntryDelete` - protocol doesn't have `didDeleteEntry`
- No timestamp-based conflict resolution

## Fix Plan

### Desktop Side (main.rs)
Change:
```rust
let vault = Arc::new(RwLock::new(Vec::<VaultEntry>::new()));
let vault_clone = vault.clone();
sync_tcp::TcpSyncServer::start(vault_clone, 7878)
```
To:
```rust
let vault = Arc::new(RwLock::new(Vec::<VaultEntry>::new()));
let vault_clone = vault.clone();
let app_state = AppState { vault: Mutex::new(None) };  // Will be replaced by managed state
sync_tcp::TcpSyncServer::start_with_state(vault_clone, app_state, 7878)
```

Actually, AppState needs to be shared between Tauri and sync_tcp. The issue is main.rs creates AppState for Tauri but sync_tcp::start_with_state takes a separate app_state. Need to architect this properly.

### Better Fix: Sync entries from AppState to sync_tcp vault on each Tauri command
Instead of syncing vault when client connects, sync on every vault operation:
- After `unlock_vault`: Copy entries to sync_tcp vault
- After `add_entry`, `update_entry`, `delete_entry`: Update sync_tcp vault
- sync_tcp needs a reference to AppState's vault lock

### iOS Side (VaultManager.swift)
Implement actual merge logic in `syncService(_:didReceiveEntries:)`:
- Compare `modifiedAt` timestamps
- Add new entries, update existing if desktop is newer
- Add delegate method `syncService(_:didDeleteEntry:)` for deletions
